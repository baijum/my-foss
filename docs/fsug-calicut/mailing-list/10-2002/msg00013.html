There are two kinds of terminals.
1).
The ones connected to a serial port ( COM1 , COM2 etc). They are
generally called dump terminals.
8 and 16 port serial terminal cards used to be in the market for this
purpose.
If you want to try this read the documentation on programs like mgetty.


2)
Terminals connected  using ethernet. Here the terminal should be capable
of running an OS and do a remote login
to the server. You can connect to a redhat server using the programs
like telnet or rlogin.
Using the LTS software one can have diskless graphics terminals
connected to a server.
