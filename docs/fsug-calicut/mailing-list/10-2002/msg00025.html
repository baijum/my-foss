Download the latest versions of glib,atk,pango and gtk+ from 
ftp.gtk.org and install this in the same order.
(try CVS versions also, visit http://developer.gnome.org/tools/cvs.html for 
 more information about howto use GNOME CVS)

If net connection is slow, try to get Redhat 8.0 it has pango version 
with Malayalam OTF support
Only XFT backend module is available for Malayalam, so
you have to set GDK_USE_XFT=1 
plain X backend module for TTF is available at  
sf.net/projects/malayalamlinux CVS tree 
(You can move this to our new CVS repository, also 
we can move locale,XKB file and all other project file to CVS
The OTF fonts can dump into XML by using TTX then it can 
check in to CVS)

If fontconfig (new font configuration tool for X) is not in RedHat CD
you have to get it anyway to use OTF fonts in GNOME 2.x

Join the smc-devel@xxxxxxxxxx and smc-users@xxxxxxxxxx list.

In KDE, you cannot get proper rendering for Malayalam glyphs,
OpenType layout support is not added.

Yudit (http://www.yudit.org) has full support for Malayalam.
use MalOtf.ttf font, just copy this
font to $HOME/.yudit/fonts
Add an MalOtf.ttf to TrueType font list in the file 
$HOME/.yudit/yudit.properties
You can save this as PS files.
If you want a High quality OpenType font
for printing purpose.
just drop a line to Raj Kumar <raj2569@xxxxxxxxx>
This font is going to release under GPL.
