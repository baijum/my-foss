>    As seen in it's release notes,

> - It's a Central Govt. project, funded by Ministry of Information

>
> - It has OpenType font support
>
> - It has Built in Indic Script Shaping Engine. This engine is being used
> by X server as well as Xlib library. This shaping engine is a shared
> object (.so file) and can also be used by client programs.
>    It is capable for script breaking, syllable breaking, reordering,
> etc. which is needed for complex Indic script processing.
>
> - The IndiX system has built in Indic locale with Unicode (UTF-8)
> support. This UTF-8 locale can be used by other libraries or applications.

indiX is really a working solution for Indian languages based on 
unicode,OpenType etc.. The last month's PC Quest has got indiX in their CD. 
(I didn't tried it yet)

But there is a major drawback for their solution they are braking the 
architecture X Window system, by implementing rendering of fonts 
at X level (That's why Xfree86 project didn't accepted their patches). Infact 
these kind of things should be handled at toolikt level
(GTK+,Qt etc). Now GTK+ has alomost suceeded in this, Recently 
Eric Mader has added OpenType layout support for Indian languages.
(Malayalam has got some problems in this implementation, will correct it)
I think next major release of Qt will be having OTF and Indic OpenType layout 
support. 

Now Malayalam is working perfectly in GNOME without OTF support (only TTF)
by using the pango modules by Arun M and N.V Shaji.
